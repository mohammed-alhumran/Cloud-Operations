# Cloud Operations — Lesson 1 Log Parser

Overview

This small repo contains Lesson 1 from a Cloud Operations exercise: safely parsing lines from log files. The parsing function handles trimming, splitting by `|`, and ignores malformed lines.

**Repository Files**

- `task1_parse_single.py`: implement `parse_log_line(line)` and run four test cases.
- `task2_parse_list.py`: parse a Python list of lines and skip invalid ones.
- `task3_parse_file.py`: read `sample_logs.txt` line-by-line and print the first 5 valid parsed tuples.
- `task4_report_invalids.py`: count total, valid, and invalid-format lines and print counts.
- `task5_export_parsed_json.py`: parse valid lines and write `parsed_logs.json`.
- `sample_logs.txt`: example log lines (provided).
- `parsed_logs.json`: generated by `task5_export_parsed_json.py` (output).

Tasks Covered

1. Parse a single line into `(timestamp, level, service, message)` or return `None` for invalid lines.
2. Parse a list of lines, collecting only valid tuples.
3. Parse a file line-by-line and show the first 5 valid entries.
4. Report counts: total, valid, invalid format lines (exact output format required).
5. Export parsed results to `parsed_logs.json` as an array of objects.

Usage

Run each script with Python 3 from the repo root:

```bash
python3 task1_parse_single.py
python3 task2_parse_list.py
python3 task3_parse_file.py
python3 task4_report_invalids.py
python3 task5_export_parsed_json.py
```

Expected outputs

- `task1_parse_single.py`: prints results of 4 unit-like test cases (two valid, two invalid).
- `task2_parse_list.py`: prints parsed tuples for valid list entries.
- `task3_parse_file.py`: prints the first five valid parsed tuples from `sample_logs.txt`.
- `task4_report_invalids.py`: prints the counts exactly like:

```
Total lines: X
Valid lines: Y
Invalid format lines: Z
```

- `task5_export_parsed_json.py`: writes `parsed_logs.json` containing an array of objects like:

```json
[
  {"timestamp":"...","level":"...","service":"...","message":"..."},
  ...
]
```

Implementation notes / Tips

- All parsing functions call `line = line.strip()` first.
- Empty lines return `None`.
- Lines are split on `|`, each field is stripped, and exactly 4 fields are required.
- Lesson 1 does not validate log level values; it only validates the 4-field format.
